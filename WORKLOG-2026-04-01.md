## 风险与注意事项

- `template-data.js`、`template-adapters.js` 当前是轻量 loader；以后新增 manifest 或新适配器时，仍需要在 loader 中补一行脚本列表，但不需要再修改编辑页 / 工作台 / 预览页核心分发文件。
- 当前三个正式模板都已迁移到平台注册中心；如果后续继续补功能，应该优先继续走适配器接口，不要再回到核心页硬编码分支。
- 隐藏示例模板默认不会出现在工作台入口；如果要人工验证，可直接访问：
  - `template.html?book=demo-price`
  - `template.html?book=demo-note`
  - `preview.html?book=demo-note`
- 由于当前 WSL 环境中 `npm` / `node_modules` 的路径兼容有额外限制，本次已把服务端和前端接入代码完整落进仓库，但本地实际运行建议直接在目标服务器或稳定的 Node 环境里执行 `npm install && npm start` 验证。
